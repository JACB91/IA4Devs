## 2. Descarga el repositorio base de Github

En cada sesión dejaremos un repositorio base en GitHub donde encontrarás:

- 
Carpetas para cada ejercicio.

- 
Instrucciones completas en un README.md.

- 
Una semilla de proyecto para facilitarte el trabajo o darte pistas de lo que esperamos recibir.

Para este primer tema, utiliza el siguiente repositorio:

[**AI4Devs-intro**](https://github.com/LIDR-academy/AI4Devs-intro-2604) 

El ejercicio es reverse-string. En esa carpeta encontrarás todo lo necesario; por ahora puedes ignorar el resto.

Para poder hacer el Pull Request al terminar, y que podamos evaluar e integrar tu solución de forma segura, recuerda que debes hacer un **fork** del repositorio. Si es tu primera vez, encontrarás más instrucciones al final de esta sección.

## **3. Realiza el ejercicio**

Aunque las instrucciones también están en el README.md, te las dejamos aquí. Léelas con atención:

- 

Crea una **página web** con lógica en **JavaScript** que **invierta el orden de una cadena de texto**.

Ejemplo: si introduzco AI4Devs, la aplicación debe devolver sveD4IA.

- 
Apóyate en la **semilla HTML + JS** que proporcionamos dentro de la carpeta template.

- 

**Usa un chatbot de IA** (como ChatGPT, Gemini o Claude) para desarrollar la solución.

**No** utilices un asistente de código integrado en un IDE (como GitHub Copilot).

Para la entrega, realiza un **Pull Request** que incluya:

- 
Una carpeta copiada desde template, con el nombre solved-iniciales (por ejemplo, reversestring-ARM).

- 

Esta carpeta debe incluir:

El **código generado**.

- 

Un archivo prompts.md con:

El **prompt** (o los prompts) utilizados.

- 
El **chatbot** utilizado.

- 
Si has usado más de un prompt hasta llegar a una solución adecuada, añade **todos** en orden.

- 
Además, incluye el **prompt final** en el **comentario del Pull Request**.

🚀 **SIGUIENTE NIVEL (Opcional)**

Si has cumplido el objetivo del reto sin problemas y quieres subir un poco más el nivel, implementa además lo siguiente:

- 
El **botón** debe aparecer solo cuando haya texto suficiente, es decir, **más de 3 letras**.

- 
La cadena invertida debe actualizarse **en tiempo real**, de forma que **no dependa del botón**.

## 4. Entrega el ejercicio

Como se indica en la parte final de las instrucciones, esperamos tu entrega como un **Pull Request** en el repositorio.

Si por algún motivo el Pull Request falla o no puedes completarlo:

- 
Envía el proyecto en **archivo .zip** por correo, o comparte un **enlace a Drive o WeTransfer**, dirigido al **Teaching Assistant**.

Si tienes dudas sobre el ejercicio:

- 
Consúltalas en el **grupo de WhatsApp**, para que podamos apoyarte lo antes posible y para que otr@s compañer@s también se beneficien de las respuestas.

Enviar tu solución mediante un **Pull Request**.
